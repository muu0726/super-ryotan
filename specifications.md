# SUPER RYO-TAN ゲーム仕様書

本作は、キャラクター「Ryo-tan」を操作して全10ステージを攻略する、ブラウザ向け2D横スクロールアクションゲームです。
Vite + Vanilla JS で開発され、高速な読み込みと滑らかな描画性能を両立したネオン調のプレミアムデザインを採用しています。

---

## 1. 基本システム・アーキテクチャ

*   **描画方式**: HTML5 Canvas API (解像度 $800 \times 450$ ピクセルの 16:9 固定スケール)
*   **ゲームループ**: **蓄積型固定タイムステップ (Fixed Timestep @ 60FPS / 16.67ms)**
    *   デバイスの画面リフレッシュレート（60Hz、120Hz等）に関わらず、壁のすり抜けや挙動の乱れを防ぐ完全決定論的な物理シミュレーション。
    *   レンダリング時は、物理更新フレーム間の残余時間（$\alpha$）を用いた**線形補間 (Lerp)** により、非常に滑らかなアニメーションを実現。
*   **アセット読み込み**: 
    *   キャラクターのポーズ切り出しは、透明化処理済みの [character.png](file:///c:/制作物/super-ryotan/public/character.png) より座標指定で切り出し。
    *   音響アセットは読み込まず、**Web Audio API** のオシレータによる波形合成（シンセサイザー）で全ての効果音を軽量に生成。
*   **レスポンシブ対応**: CSS 3D Transform によりゲーム画面全体をアスペクト比を維持して自動拡大縮小。縦画面の時は自動で警告オーバーレイを表示。

## 2. 物理演算仕様

ゲーム内のキャラクターや敵の移動・衝突は、AABB（軸平行バウンディングボックス）を用いた格子単位の衝突解決を行っています。1ブロックのサイズ（`TILE`）は **$32 \text{ px}$** です。

| 物理パラメータ | 定数値 (px/frame) | 実際の物理挙動への影響 |
| :--- | :--- | :--- |
| **最大歩行速度 (`V_WALK_MAX`)** | `2.5` | 方向キーのみの入力時の上限速度 |
| **最大ダッシュ速度 (`V_DASH_MAX`)** | `4.5` | ダッシュボタンホールド/トグル時の上限速度 |
| **水平加速度 (`A_X`)** | `0.1` | 入力開始からトップスピードへ到達する滑らかさ |
| **ブレーキ減速度 (`D_X`)** | `0.30` | 移動方向と逆方向にキーを入力した時の減速慣性 |
| **空気/床面摩擦係数 (`F_X`)** | `0.15` | キー入力を離した際に滑りながら減速する慣性 |
| **初期ジャンプ初速 (`JUMP_V`)** | `-6.7` | ジャンプボタン押下時の瞬間的な上向き速度 |
| **走行時ジャンプ初速 (`JUMP_V_DASH`)** | `-7.5` | 水平速度が `3.5` 以上の時の強化初速（SMB準拠: 走るほど高く跳べる） |
| **上昇中（長押し）重力 (`G_RISE`)** | `0.28` | ジャンプボタンを押し続けている上昇中の適用重力（低重力） |
| **通常/落下中重力 (`G_FALL`)** | `0.56` | ボタンを離した上昇中、または下降中の適用重力（高重力） |
| **最大落下終端速度 (`TERMINAL_V`)** | `8.0` | 落下速度の上限値（すり抜けを防止する安全制限） |

### 可変ジャンプ仕様
プレイヤーの上昇速度が負の時、ジャンプボタンが離されると上昇速度が半減（`vy *= 0.5`）され、重力が通常の2倍（`G_FALL`）に切り替わります。これにより、ボタンの押下時間でジャンプ高度を細かくコントロールできます。

### 操作の手触り（モダンQoL）
*   **走行連動ジャンプ**: ジャンプ踏切時の水平速度が `3.5 px/frame` 以上なら初速が `-7.5` に強化され、ダッシュジャンプはより高く・遠くへ届きます（実際のマリオの「速度でジャンプ力が伸びる」仕様に準拠）。
*   **コヨーテタイム (`COYOTE_FRAMES = 5`)**: 崖から足が離れた後も5フレームの間はジャンプ入力を受け付け、「踏み切ったつもりが落ちた」理不尽さを軽減します。
*   **ジャンプ先行入力 (`JUMP_BUFFER_FRAMES = 6`)**: 着地の最大6フレーム前からのジャンプ入力を記憶し、着地フレームで自動的にジャンプが発動します。連続ジャンプの操作感が向上します。

## 3. 操作仕様

PCとスマートフォンの両方に最適化されたインターフェースをサポートしています。

### PC（キーボード）
*   **左右移動**: `A` / `D` または `←` / `→`
*   **ジャンプ**: `Space` または `Z`
*   **ダッシュ**: `Shift` または `X` （**ホールド式**: 押している間だけ走り、離すと歩行）

### スマートフォン（タッチパネル）
画面下部に仮想の半透明マルチタッチコントローラーを描画します。
*   **十字キー（左 / 右）**: タッチエリア拡張（ヒット判定を1.5倍に拡大）
*   **Aボタン**: ジャンプ
*   **Bボタン**: ダッシュ（**トグル式**: 1回タップでON、もう1回タップでOFF。快適な両手操作のため）

## 4. タイル・オブジェクト仕様

ステージはテキスト（文字グリッド）で定義されています。

*   `#` (ソリッドブロック): プレイヤーや敵が乗れる・衝突する不透過ブロック。
*   `=` (すり抜け床): 下からジャンプで通り抜け可能で、上からは着地できる一方通行のプラットフォーム。
*   `B` (レンガブロック): 下から叩くと小さく飛び跳ねる演出が入り、上にいる敵を倒せます（破壊はできません）。
*   `?` (ハテナブロック): 下から叩くとコインが上に飛び出し、獲得。叩いた後は `U` (空ブロック) に変化。
*   `M` (きのこ入りハテナブロック): 見た目は `?` と同一（SMB準拠で中身は開けるまで不明）。下から叩くとブロック上面から**スーパーきのこ**がせり上がり、`U` に変化。
*   `U` (空ブロック): ソリッドな灰色のブロック。叩くことはできません。
*   `o` (コイン): 空中に浮遊している収集要素。接触すると消滅し、獲得（`sfxCoin` SE）。
*   `H` (トゲ): タイル下半分に当たり判定があり、接触するとプレイヤーは即座にミスとなります。
*   `S` (スタート位置): プレイヤーがステージ開始時に配置される座標（読み込み後は空気タイルに置換）。
*   `G` / `g` (ゴールポール): 触れるとステージクリアとなり、ゴール演出シーケンス（下記）へ移行。

### ゴール演出シーケンス（SMB1準拠）

ポールに触れた瞬間にタイム計測が停止し、以下の4フェーズが順番に実行されます。

1.  **滑降 (slide)**: プレイヤーがポール左側に張り付き、旗と一緒にポールを滑り降ります（プレイヤー 2.4 px/frame、旗 3 px/frame）。`sfxFlag`（下降スライドホイッスル音）が再生されます。
2.  **静止 (pause)**: 着地後、約22フレームの間ポールに掴まったまま静止します。
3.  **飛び降り (hop)**: ポールの反対側（右側）へ小さくジャンプして降ります。このタイミングで `sfxClear` ファンファーレが再生され、**「COURSE CLEAR!」** の表示と、ポールを掴んだ高さに応じたボーナスコイン（最下部 +1 〜 最上部 +10。SMB1 の 100〜5000点ボーナスのコイン換算）が加算されます。
4.  **歩き去り (walk)**: 着地後、右方向へ自動歩行（2.0 px/frame）します。歩行中、空に **打ち上げ花火** が3発上がり（SMB1 のタイマー末尾 1/3/6 花火のオマージュ）、画面右端へ到達するか約140フレーム経過するとクリア画面へ遷移します。

## 5. パワーアップシステム（スーパーきのこ）

SMB1準拠の2段階パワー状態を実装しています。

*   **パワー状態**: `small`（初期状態、当たり判定 24×44px）と `super`（きのこ取得後、24×58px）。描画スケールも連動して大きくなります。
*   **スーパーきのこ**: `M` ブロックを叩くと上面からせり上がり（約0.7秒、この間は衝突なし）、その後 1.4 px/frame で自走します。壁で反転しますが、**SMB準拠で崖からはそのまま落ちます**。
*   **取得効果**: small 時はスーパー化（`sfxPowerup` 上昇アルペジオ + 「パワーアップ!」表示）。super 時に重ねて取ると **コイン +2**（SMB1のスコアボーナスに相当）。
*   **被ダメージ**: super 時に敵・トゲに触れると死なずにスモール化し（`sfxShrink` 下降音）、**120フレームの無敵時間**（点滅表示、敵・トゲとの接触無効）が付与されます。small 時の被弾は従来通りミス。穴への落下は状態に関わらずミスです。
*   **リセット**: ミス・ステージ開始時は small 状態から再スタートします。

## 6. 敵キャラクター（3タイプ）

| 記号 | タイプ | 挙動 |
| :--- | :--- | :--- |
| `E` | クリボー風 | 左右巡回。壁・**崖の端で自動反転**し穴に落ちない。踏むと潰れて撃破。 |
| `K` | ノコノコ風 | 左右巡回（SMB準拠で**崖からは落ちる**）。踏むと**甲羅化**する（下記）。 |
| `F` | パタパタ風 | 空中の基準点を中心に**上下±28pxをふわふわ浮遊**（地形・重力の影響なし）。踏むと羽を失いクリボー風の歩行型に変化し、もう一度踏むと撃破。 |

### 甲羅システム（ノコノコ風）

*   **甲羅化**: 歩行中に踏むと甲羅にこもり停止します（プレイヤーは跳ね返り）。
*   **蹴り出し**: 静止甲羅に横から触れる、または上から踏むと、プレイヤーと反対方向へ **4.5 px/frame** で滑走します（横から触れてもダメージなし、`sfxKick` SE）。
*   **連鎖撃破**: 滑走中の甲羅は接触した他の敵をすべて撃破します（火花パーティクル + SE）。
*   **滑走中の甲羅は危険**: 横から触れると被ダメージ。上から踏むと停止して静止甲羅に戻ります。
*   **復帰**: 甲羅のまま300フレーム経過すると歩行状態に復帰します（SMB準拠）。

### 共通の衝突解決

*   **上から踏んだ場合**: タイプ別の応答（撃破 / 甲羅化 / 歩行化）が発生し、プレイヤーは上空に跳ね返ります（`vy = STOMP_BOUNCE_V = -7.5`。SMB準拠でジャンプ長押しなら通常ジャンプ相当の約100px＝はてなブロックの上に乗れる高さ、長押しなしなら約50pxの小バウンド）。`sfxKick` SEが再生されます。潰れた敵は30フレーム表示後に消滅します。
*   **横または下から衝突した場合**: 被ダメージ（super時はスモール化、small時はミス）。無敵時間中は接触してもすり抜けます。

## 7. パーティクル・演出システム

臨場感と操作フィードバックを高めるため、軽量なカスタムパーティクルシステムを搭載しています。

1.  **コインポップアップ**: ハテナブロックを叩いた際、コインが回転しながら上に飛び出してから消える演出。
2.  **走行煙**: プレイヤーがダッシュに近い高速移動をしている間、足元から後方へ向けて定期的に小さな煙の粒子を放出。
3.  **急制動（スリップ）煙**: 逆方向キーを入力してプレイヤーがキキーッと急減速している間（スリップ状態）、摩擦を示す密集した煙を足元から放出。
4.  **着地煙**: 空中から地面に着地した瞬間、プレイヤーの足元から左右両方向に向けて煙が広がるように放出。
5.  **打ち上げ花火**: ゴール演出の歩き去りフェーズ中、空中の1点から18方向へカラフルな火花が放射状に広がる花火が3発打ち上がる（`sfxFirework` SE付き）。
6.  **ボーナステキスト**: ゴールポールを掴んだ高さに応じた「コイン +N」のテキストがゆっくり上昇しながらフェードアウト。

## 8. タイムアタック＆セーブ機能

*   **進捗セーブ**: クリアしたステージ数に基づき、解放された先のステージ情報が `localStorage` (`super-ryotan-progress-v1`) に自動保存され、次回起動時もステージ選択から再開可能。
*   **タイムアタック**:
    *   ステージのロードからクリアまでのプレイ時間を $60 \text{ Hz}$ 精度のフレーム数で計測（0.1秒単位でHUDに表示）。
    *   クリア時、過去の記録よりも早いタイムであれば `localStorage` (`super-ryotan-time-v1-<ステージ番号>`) に上書き保存。
    *   自己ベストを更新すると、クリア画面に **「🏆 NEW RECORD! 🏆」** のゴールドトロフィーバッジが表示されます。
    *   ステージ選択画面のグリッドボタンには、ステージ解放状態に加えてベストクリアタイムが `⏱12.4s` のようにバッジとして表示されます。

## 9. 音響合成仕様 (Web Audio API)

外部ファイルを使用せず、コード上でシンセサイズされたレトロな8bit効果音です。

*   `sfxJump` (ジャンプ音): $150 \text{ Hz}$ から $650 \text{ Hz}$ へ 0.13秒間で急激に周波数を引き上げるサイン波（ピョーン音）。
*   `sfxCoin` (コイン音): $987 \text{ Hz}$ (B5) と $1318 \text{ Hz}$ (E6) を 0.08秒間隔で鳴らす矩形波のアルペジオ（チャリン音）。
*   `sfxBump` (壁ぶつかり音): 低域 $120 \text{ Hz}$ を三角波で極めて短時間だけ鳴らす鈍い衝突音（ドッ音）。
*   `sfxBlock` (ハテナブロック音): $261 \text{ Hz}$ (C4) から $880 \text{ Hz}$ (A5) までのアルペジオにマイルドなディケイを付加（ピコッ音）。
*   `sfxDeath` (プレイヤー死亡音): $600 \text{ Hz}$ 付近から不協和音を伴いながら下方向へスウィープし、最後に下降スケールを奏でる矩形波。
*   `sfxClear` (ファンファーレ): ド・ミ・ソ・ドのメジャーコード上昇アルペジオに続いて $1046 \text{ Hz}$ (C6) の長音を伸ばす、コード合成による勝利ジングル。
*   `sfxKick` (敵踏みつけ音): 超低域 $80 \text{ Hz}$ から $30 \text{ Hz}$ に向けて急激に下降する三角波を重ねた太い打撃音（ボコッ音）。
*   `sfxPowerup` (パワーアップ音): C4 から C6 まで駆け上がる7音の矩形波アルペジオ（SMB1のきのこ取得音風）。
*   `sfxShrink` (縮小音): $784 \text{ Hz}$ から $247 \text{ Hz}$ へ5段階で下降するステップ音（SMB1のダメージ縮小音風）。
*   `sfxFlag` (ゴールポール滑降音): $1568 \text{ Hz}$ から $196 \text{ Hz}$ へ約0.9秒かけて滑らかに下降する矩形波+三角波（スライドホイッスル風）。
*   `sfxFirework` (花火音): ローパスノイズの破裂音に低域へ落ちる三角波の余韻を重ねた「ドーン」音。
